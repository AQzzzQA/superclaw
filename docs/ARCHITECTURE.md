## 系统概述

SuperClaw 是一个融合 OpenClaw 的易用性和 LemClaw 的安全性 + IronClaw 的 Echo Skills 特性的下一代智能体平台。

### 核心特性

- 🚀 **双网关兼容** - OpenClaw Gateway（WebSocket） + LemClaw Gateway（HTTP）
- 🛡️ **企业级安全** - WebAssembly 沙箱隔离、令牌白名单、多层防护
- 🤖 **智能体编排** - 并行/串行协作、任务分解、结果汇总
- 🔌 **灵活插件** - WASM 插件、多模型支持
- 📊 **自动化能力** - Echo Skills 自动修复、代码扫描、文档生成

### 设计目标

- **高性能** - 异步架构，低延迟响应
- **高可用** - 双网关自动切换，故障转移
- **可扩展** - 插件化架构，易于扩展
- **易维护** - 模块化设计，清晰的代码结构
- **企业级** - 安全、监控、日志、审计

## 模块设计

### 1. Gateway 模块

#### WebSocket Gateway

**职责**：
- 接收客户端 WebSocket 连接
- 实时推送消息
- 连接池管理

**组件**：
- `ConnectionManager` - 连接管理器
- `WebSocketHandler` - 消息处理器
- `Broadcaster` - 消息广播器

**API**：
```
GET  /ws                    - WebSocket 连接
POST /api/websocket/message - WebSocket API 消息
GET  /api/websocket/connections - 列出连接
```

#### HTTP Gateway

**职责**：
- 提供 RESTful API
- HTTP 请求处理
- 响应序列化

**组件**：
- `LemClawProxy` - LemClaw 代理
- `OpenClawProxy` - OpenClaw 代理
- `RequestHandler` - 请求处理器

**API**：
```
GET  /health        - 健康检查
POST /api/agent     - 发送消息
GET  /api/status     - 网关状态
```

### 2. Echo Skills 模块

#### CodeScanner（代码扫描器）

**职责**：
- 扫描代码库
- 检测代码问题
- 生成问题报告

**工具**：
- flake8 - 代码规范检查
- mypy - 类型检查
- bandit - 安全漏洞扫描

**API**：
```
POST /api/skills/scan     - 扫描代码库
POST /api/skills/fix      - 自动修复问题
```

#### AutoFixer（自动修复器）

**职责**：
- 自动格式化代码
- 修复低风险问题
- 生成修复报告

**工具**：
- black - 代码格式化
- isort - import 排序
- autoflake - 移除未使用 import

#### SmartGenerator（智能生成器）

**职责**：
- 生成文档
- 生成配置文件
- 生成许可证

**功能**：
- 自动生成 CHANGELOG.md
- 自动生成 LICENSE
- 自动生成 .env.example

### 3. Agent Orchestrator 模块

**职责**：
- 管理多个智能体
- 任务调度
- 结果汇总

**组件**：
- `AgentPool` - 智能体池
- `TaskScheduler` - 任务调度器
- `ResultAggregator` - 结果汇总器

**功能**：
- 并行执行 - 多个智能体同时工作
- 串行执行 - 按顺序执行任务
- 任务分解 - 复杂任务自动拆解

## 数据流

### 消息流程

```
客户端
  │
  ├─ WebSocket: 实时连接
  │   ├─ 1. 建立 WebSocket 连接
  │   ├─ 2. 订阅消息频道
  │   ├─ 3. 接收实时消息
  │   └─ 4. 发送消息
  │
  └─ HTTP: RESTful API
      ├─ 1. 发送 HTTP 请求
      ├─ 2. 接收 JSON 响应
      └─ 3. 处理错误
      │
      ↓
SuperClaw Gateway
  │
  ├─ 路由: 路由请求
  ├─ 认证: 验证授权
  ├─ 代理: 转发到后端
  │   ├─ LemClaw Gateway (HTTP)
  │   └─ OpenClaw Gateway (WebSocket)
  ├─ 切换: 故障自动切换
  └─ 响应: 返回结果
      │
      ↓
AI 服务 (OpenClaw/LemClaw)
  │
  ├─ 处理消息
  ├─ 调用 AI 模型
  └─ 返回回复
      │
      ↓
SuperClaw Gateway
  │
  └─ 广播: 推送到客户端
      │
      ↓
客户端
```

### 数据存储流程

```
请求 → 验证授权 → 缓存查询 → 数据库查询 → 缓存更新 → 响应
```

## 安全架构

### 1. 多层防护

```
┌─────────────────────────────────────────┐
│  Layer 1: 网络层                │
│  - HTTPS/TLS 加密               │
│  - IP 白名单                      │
│  - 速率限制                       │
├─────────────────────────────────────────┤
│  Layer 2: 应用层                │
│  - 授权码验证                    │
│  - 会话管理                      │
│  - 权限控制                      │
├─────────────────────────────────────────┤
│  Layer 3: 数据层                │
│  - 数据加密存储                  │
│  - 敏感信息脱敏                  │
│  - 审计日志                      │
└─────────────────────────────────────────┘
```

### 2. 安全措施

- **授权码系统** - 每次请求验证授权码
- **令牌白名单** - 只允许白名单令牌访问
- **审计日志** - 记录所有敏感操作
- **WebAssembly 沙箱** - 插件隔离执行
