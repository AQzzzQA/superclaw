## 🚀 快速开始

### 什么是 OpenClaw？

OpenClaw 是一个**智能体编排平台**，让你的 AI 助手能够：
- 🤖 **多平台对话**：飞书、微信、WhatsApp、QQ 等
- 🧩 **技能扩展**：像安装插件一样添加新能力
- 🔄 **自动化任务**：定时提醒、数据抓取、报告生成
- 🤝 **多智能体协作**：让不同的 AI 协同工作

### 3 分钟上手

1. **在飞书中 @AI 助手**
2. **发送指令或自然语言请求**
3. **AI 自动执行并返回结果**

就这么简单！

## 🪶 飞书集成配置

### 第一步：创建飞书自建应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 进入「管理后台」→「应用管理」→「创建企业自建应用」
3. 填写应用信息：
   - 应用名称：`AI 智能助手`（或自定义）
   - 应用描述：`OpenClaw 智能体编排平台`

### 第二步：配置权限

打开**全部权限**，确保以下权限已开启：

| 权限类别 | 必需权限 | 说明 |
|---------|---------|------|
| **消息** | `im:message` | 发送/接收消息 |
| **消息** | `im:message:group_at_msg` | 群组 @ 消息 |
| **文档** | `drive:drive:readonly` | 读取云盘文件 |
| **文档** | `drive:drive:write` | 写入云盘文件 |
| **文档** | `docx:document:readonly` | 读取文档 |
| **文档** | `docx:document:write` | 编辑文档 |
| **知识库** | `wiki:wiki:readonly` | 读取知识库 |
| **日历** | `calendar:calendar:readonly` | 读取日历 |
| **联系人** | `contact:user.base:readonly` | 读取用户信息 |
| **机器人** | `im:chat` | 聊天权限 |

### 第三步：获取凭证

1. 进入应用「凭证与基础信息」
2. 复制以下信息：
   ```
   App ID: cli_xxxxxxxxxxxxxx
   App Secret: xxxxxxxxxxxxxxxx
   ```

### 第四步：配置 OpenClaw

编辑 OpenClaw 配置文件 `openclaw.json`：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_xxxxxxxxxxxxxx",
      "appSecret": "xxxxxxxxxxxxxxxx",
      "encryptKey": "（可选）加密密钥",
      "verificationToken": "（可选）验证令牌"
    }
  }
}
```

### 第五步：启动服务

```bash
# 启动 OpenClaw
openclaw gateway start

# 查看状态
openclaw gateway status
```

### 第六步：测试连接

在飞书中找到你的 AI 应用，发送：

```
hello
```

如果看到回复，说明配置成功！🎉

## 🧩 核心技能介绍

OpenClaw 的能力通过**技能**扩展，以下是公司常用的核心技能：

### 📄 飞书文档技能 (feishu-doc)

**功能**：读写飞书文档、创建文档、格式化内容

**典型场景**：
- 自动生成周报、月报
- 批量修改文档格式
- 提取文档关键信息

**示例指令**：
```
读取文档 https://xxxxx.feishu.cn/docx/xxxxx
```

### 📁 飞书云盘技能 (feishu-drive)

**功能**：管理飞书云空间，上传、下载、搜索文件

**典型场景**：
- 批量整理云盘文件
- 自动归档旧文件
- 查找特定文件

**示例指令**：
```
搜索云盘中所有包含"合同"的文件
```

### 🔐 飞书权限技能 (feishu-perm)

**功能**：管理文档和文件的分享权限、协作者管理

**典型场景**：
- 批量设置文档权限
- 审查敏感文档的访问者
- 自动化权限管理

**示例指令**：
```
把文档 https://xxxxx.feishu.cn/docx/xxxxx 分享给张三，权限为可编辑
```

### 📚 飞书知识库技能 (feishu-wiki)

**功能**：导航和搜索飞书知识库

**典型场景**：
- 快速查找知识库内容
- 自动更新知识库
- 知识库内容分析

**示例指令**：
```
在知识库中搜索"AI 使用指南"
```

### 🕒 定时提醒技能 (qqbot-cron)

**功能**：设置一次性提醒、周期性任务、自动降级确保送达

**典型场景**：
- 会议提醒
- 任务截止提醒
- 定期报告

**示例指令**：
```
下午 3 点提醒我开会
```

### 🖼️ 图片发送技能 (qqbot-media)

**功能**：发送本地图片到聊天

**典型场景**：
- 发送图表截图
- 发送设计稿
- 发送数据可视化结果

**示例指令**：
```
生成柱状图并显示出来
```

### 🔍 网页搜索技能 (web_search)

**功能**：使用 Brave Search 进行网络搜索

**典型场景**：
- 查找最新资讯
- 技术问题调研
- 竞品分析

**示例指令**：
```
搜索"AI 智能体最新进展"
```

### 📝 GitHub 技能 (github)

**功能**：与 GitHub 交互，管理 issues、PR、CI 运行

**典型场景**：
- 创建 Issue 跟踪 bug
- 审查 Pull Request
- 查看 CI 状态

**示例指令**：
```
查看 PR #123 的状态
```

### 🌐 网页自动化技能 (agent-browser)

**功能**：自动化网页操作，点击、填表、截图

**典型场景**：
- 自动化数据抓取
- 表单自动填写
- 定时截图监控

**示例指令**：
```
打开 https://example.com，截图发给我
```

### ⏰ 天气技能 (weather)

**功能**：获取当前天气和天气预报（无需 API Key）

**典型场景**：
- 每日天气查询
- 出行建议
- 活动规划

**示例指令**：
```
查一下北京今天天气
```

## ❓ 常见问题

### Q1: AI 不回复消息怎么办？

**可能原因**：
1. OpenClaw 服务未启动
2. 飞书权限未开启
3. 网络连接问题

**解决方法**：
```bash
# 检查 OpenClaw 状态
openclaw gateway status

# 重启服务
openclaw gateway restart

# 查看日志
openclaw gateway logs
```

### Q2: 如何设置机器人权限？

**步骤**：
1. 进入飞书开放平台
2. 选择你的应用
3. 进入「权限管理」
4. 开启所需权限
5. 发布应用

### Q3: 可以同时接入多个飞书应用吗？

**可以**。在配置文件中添加多个账号：

```json
{
  "feishu": {
    "accounts": [
      {
        "id": "hr-bot",
        "appId": "cli_xxx1",
        "appSecret": "xxx1"
      },
      {
        "id": "sales-bot",
        "appId": "cli_xxx2",
        "appSecret": "xxx2"
      }
    ]
  }
}
```

### Q4: 如何保护敏感数据？

**建议**：
1. 不在 AI 对话中直接输入密码、密钥
2. 使用飞书的加密功能
3. 定期审查文档权限
4. 开启操作日志

### Q5: 定时任务不执行怎么办？

**排查步骤**：
1. 检查时区设置（确保是 Asia/Shanghai）
2. 查看日志是否有错误
3. 确认任务已正确设置
4. 检查服务器时间是否正确

### Q6: 如何查看历史消息？

**方法**：
```bash
# 查看最近的对话
openclaw sessions history --limit 10

# 导出对话记录
openclaw sessions export --format json > history.json
```

## 🎯 最佳实践

### 1. 命名规范

给 AI 应用取一个清晰的名字，方便员工识别：
- ✅ 好的命名：`AI 智能助手`、`数据小助手`、`运维机器人`
- ❌ 不好的命名：`Bot`、`机器人`、`Test`

### 2. 权限最小化原则

- 只授予必需的权限
- 定期审查权限
- 敏感操作需要二次确认

### 3. 明确指令

给 AI 发送指令时，越具体越好：
- ✅ 好的指令：「在飞书云盘中搜索所有包含"合同"且 2025 年创建的文件」
- ❌ 不好的指令：「搜索合同」

### 4. 定期备份

- 定期备份重要数据
- 导出 AI 对话记录
- 保存配置文件

### 5. 培训员工

- 收集常见问题

### 6. 监控和优化
