
## 🔐 用户认证

### API认证

DSP Platform使用JWT Token进行认证：

**步骤1: 获取Token**
```bash
curl -X POST http://43.156.131.98:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "your_username",
    "password": "your_password"
  }'
```

**响应示例**:
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer",
  "expires_in": 3600
}
```

**步骤2: 使用Token访问API**
```bash
curl http://43.156.131.98:8000/api/v1/campaigns/ \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

## 📊 API使用示例

### 1. 创建广告计划

```bash
curl -X POST http://43.156.131.98:8000/api/v1/campaigns/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "name": "春节促销活动",
    "platform": "douyin",
    "budget": 10000.00,
    "start_date": "2026-02-01",
    "end_date": "2026-02-15",
    "status": "active"
  }'
```

### 2. 查询广告计划列表

```bash
curl http://43.156.131.98:8000/api/v1/campaigns/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -G \
  --data-urlencode "page=1" \
  --data-urlencode "page_size=20" \
  --data-urlencode "status=active"
```

### 3. 上传创意素材

```bash
curl -X POST http://43.156.131.98:8000/api/v1/creatives/ \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "name=春节海报" \
  -F "type=image" \
  -F "file=@/path/to/image.jpg"
```

### 4. 创建报表

```bash
curl -X POST http://43.156.131.98:8000/api/v1/reports/ \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "name": "春节活动数据报表",
    "campaign_id": 1,
    "start_date": "2026-02-01",
    "end_date": "2026-02-15",
    "metrics": ["impressions", "clicks", "conversions"]
  }'
```

### 5. 数据回传

```bash
curl -X POST http://43.156.131.98:8000/api/v1/data/impressions/ \
  -H "Content-Type: application/json" \
  -d '{
    "campaign_id": 1,
    "creative_id": 1,
    "user_id": "user123",
    "timestamp": "2026-02-01T12:00:00",
    "device": "mobile",
    "os": "iOS",
    "platform": "douyin"
  }'
```

## 🛠️ 服务管理

### 启动服务

```bash
cd /root/.openclaw/workspace/dsp-platform
docker-compose up -d
```

### 停止服务

```bash
cd /root/.openclaw/workspace/dsp-platform
docker-compose down
```

### 重启单个服务

```bash
cd /root/.openclaw/workspace/dsp-platform
docker-compose restart [service-name]

# 示例
docker-compose restart dsp-backend
```

### 查看服务状态

```bash
cd /root/.openclaw/workspace/dsp-platform
docker-compose ps
```

### 查看服务日志

```bash
# 查看所有日志
docker-compose logs -f

# 查看单个服务日志
docker-compose logs -f [service-name]

# 查看最近100行日志
docker-compose logs --tail=100 [service-name]

# 示例
docker-compose logs -f dsp-backend
docker-compose logs --tail=50 dsp-mysql
```

## 🔧 故障排查

### 问题1: API无法访问

**症状**: 访问 http://43.156.131.98:8000 无响应

**解决方法**:
```bash
# 检查服务状态
docker ps | grep dsp-backend

# 查看服务日志
docker logs dsp-backend --tail=50

# 重启服务
docker-compose restart dsp-backend
```

### 问题2: Grafana无法访问

**症状**: 访问 http://43.156.131.98:8888 显示连接错误

**解决方法**:
```bash
# 检查Grafana服务
docker ps | grep dsp-grafana

# 查看Grafana日志
docker logs dsp-grafana --tail=50

# 重启Grafana
docker-compose restart dsp-grafana
```

### 问题3: Prometheus无数据

**症状**: Grafana仪表板显示无数据

**解决方法**:
```bash
# 检查Prometheus服务
docker ps | grep dsp-prometheus

# 查看Prometheus目标
curl http://localhost:8999/api/v1/targets

# 检查Prometheus日志
docker logs dsp-prometheus --tail=50
```

## ❓ 常见问题

### Q: 如何修改Grafana密码？
A: 首次登录后，Grafana会提示修改默认密码。

### Q: 如何配置SSL证书？
A: 在Nginx配置文件中添加SSL证书配置，并重启Nginx服务。

### Q: 如何备份数据库？
A: 使用以下命令备份MySQL数据库：
```bash
docker exec dsp-mysql mysqldump -u root -p dsp_platform > backup.sql
```

### Q: 如何查看服务端口？
A: 运行以下命令查看所有端口：
```bash
docker ps --format "table {{.Names}}\t{{.Ports}}"
```
