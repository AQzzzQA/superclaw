## 可能原因

### 1. 云服务器安全组未开放端口9000 ⚠️

这是最可能的原因。需要在云服务器控制台开放端口9000。

**配置方法**：
- 登录云服务器控制台
- 找到"安全组"或"安全组规则"
- 添加入站规则：
  - 端口：9000
  - 协议：TCP
  - 授权对象：0.0.0.0/0
- 保存并等待1-5分钟生效

### 2. 防火墙规则阻止 ⚠️

检查防火墙是否阻止了端口9000。

```bash
firewall-cmd --list-all
```

如果没有开放端口9000，执行：

```bash
firewall-cmd --permanent --add-port=9000/tcp
firewall-cmd --reload
```

### 3. 浏览器缓存问题

尝试清除浏览器缓存或使用无痕模式访问。

### 4. 网络问题

检查你的电脑是否可以访问该IP：

```bash
# 在你的电脑上执行
ping 43.156.131.98
telnet 43.156.131.98 9000
```

## 解决方案

### 方案1：开放云服务器安全组（推荐）

这是最可能的原因。请按照以下步骤操作：

#### 阿里云
1. 登录阿里云控制台
2. 进入"云服务器ECS"
3. 选择你的实例
4. 点击"安全组"
5. 点击"配置规则"
6. 添加入站规则：
   - 授权策略：允许
   - 协议类型：TCP
   - 端口范围：9000/9000
   - 授权对象：0.0.0.0/0
7. 保存

#### 腾讯云
1. 登录腾讯云控制台
2. 进入"云服务器"
3. 选择你的实例
4. 点击"安全组"
5. 点击"修改规则"
6. 添加入站规则：
   - 协议端口：TCP:9000
   - 来源：0.0.0.0/0
   - 策略：允许
7. 保存

#### 其他云服务商
类似操作，找到安全组设置，开放端口9000。

### 方案2：开放服务器防火墙

如果云服务器安全组已配置，但仍无法访问，检查服务器防火墙：

```bash
# 检查防火墙状态
firewall-cmd --list-all

# 开放端口9000
firewall-cmd --permanent --add-port=9000/tcp
firewall-cmd --reload
```

### 方案3：测试端口连通性

在你的电脑上执行以下命令测试端口是否开放：

```bash
# Windows
telnet 43.156.131.98 9000

# Linux/Mac
nc -zv 43.156.131.98 9000
```

如果端口开放，应该能看到连接成功的提示。

## 替代访问方案

如果暂时无法打开端口9000，可以尝试以下替代方案：

### 方案1：使用Nginx反向代理

通过Nginx访问Prometheus（端口8080）：

访问地址：`http://43.156.131.98:8080/prometheus/`

### 方案2：SSH端口转发

在你的电脑上执行：

```bash
ssh -L 9000:localhost:9000 root@43.156.131.98
```

然后在浏览器访问：`http://localhost:9000`

### 方案3：临时更改端口

将Prometheus端口改到其他端口（如果云服务器限制）：

编辑 `docker-compose.yml`：

```yaml
prometheus:
  ports:
    - "0.0.0.0:8001:9090"  # 改到8001
```

重启服务：

```bash
cd /root/.openclaw/workspace/dsp-platform
docker-compose restart prometheus
```

访问地址：`http://43.156.131.98:8001`
