## 3. 服务详解

### 3.1 核心应用服务

#### 3.1.1 后端服务 (dsp-backend)

**职责**:
- 提供RESTful API
- 业务逻辑处理
- 数据库访问
- 缓存管理

**配置**:
- 端口: 8000
- 健康检查: /health
- 指标: /metrics
- 自动重启: unless-stopped

**环境变量**:
```yaml
DATABASE_URL=mysql://dsp_user:${MYSQL_PASSWORD}@dsp-mysql:3306/dsp_db
REDIS_URL=redis://dsp-redis:6379/0
CELERY_BROKER_URL=redis://dsp-redis:6379/0
ENVIRONMENT=${ENVIRONMENT:-development}
JWT_SECRET=${JWT_SECRET}
SENTRY_DSN=${SENTRY_DSN}
```

#### 3.1.2 前端服务 (dsp-frontend)

**职责**:
- 用户界面渲染
- 客户端路由
- API调用

**配置**:
- 端口: 3000
- 健康检查: /
- 自动重启: unless-stopped

**环境变量**:
```yaml
REACT_APP_API_URL=https://${DOMAIN_NAME}/api
ENVIRONMENT=${ENVIRONMENT:-development}
```

#### 3.1.3 Celery Worker

**职责**:
- 异步任务处理
- 定时任务执行
- 消息队列消费

**配置**:
- 启动命令: `celery -A app.celery_app worker --loglevel=info`
- 自动重启: unless-stopped

#### 3.1.4 Celery Beat

**职责**:
- 定时任务调度
- 周期性任务管理

**配置**:
- 启动命令: `celery -A app.celery_app beat --loglevel=info`
- 自动重启: unless-stopped

### 3.2 数据存储服务

#### 3.2.1 MySQL (dsp-mysql)

**职责**:
- 关系型数据存储
- 事务管理
- 数据持久化

**配置**:
- 版本: 8.0
- 端口: 3306（内部）
- 数据卷: mysql-data
- 字符集: utf8mb4

**数据库名**: dsp_db
**用户名**: dsp_user
**密码**: ${MYSQL_PASSWORD}

#### 3.2.2 Redis (dsp-redis)

**职责**:
- 缓存存储
- 会话管理
- 消息队列

**配置**:
- 版本: 7-alpine
- 端口: 6379（内部）
- 数据卷: redis-data
- 持久化: AOF
- 密码: ${REDIS_PASSWORD}

### 3.3 反向代理服务

#### 3.3.1 Nginx (dsp-nginx)

**职责**:
- HTTP/HTTPS代理
- 负载均衡
- 静态资源服务
- SSL终结

**配置**:
- 版本: 1.24-alpine
- 端口: 80, 443
- 工作进程: auto
- 连接数: 1024

**特性**:
- Gzip压缩
- 限流保护
- 安全头部
- WebSocket支持
- Let's Encrypt自动续期

## 4. CI/CD流水线

### 4.1 工作流阶段

```
触发条件: push to main/develop, pull_request, release
    │
    ▼
┌─────────────────────────────────────────┐
│  1. 代码质量检查 (lint)                 │
│     - Python: Black, isort, Flake8, mypy│
│     - JS/TS: ESLint, Prettier           │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  2. 安全扫描 (security)                 │
│     - Trivy漏洞扫描                     │
│     - Snyk依赖扫描                      │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  3. 单元测试 (test)                     │
│     - 后端: pytest + coverage           │
│     - 前端: Jest + coverage             │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  4. 构建镜像 (build)                    │
│     - 后端Docker镜像                    │
│     - 前端Docker镜像                    │
│     - 推送到GHCR                        │
└─────────────────────────────────────────┘
    │
    ├─────────────────────┬───────────────┐
    ▼                     ▼               ▼
┌─────────────┐    ┌─────────────┐  ┌─────────────┐
│deploy-staging│    │deploy-prod  │  │performance  │
│(develop分支) │    │(main分支)   │  │(性能测试)   │
└─────────────┘    └─────────────┘  └─────────────┘
```

### 4.2 部署策略

#### 4.2.1 Staging环境

**触发条件**: 推送到develop分支

**部署步骤**:
1. SSH连接到staging服务器
2. 拉取最新代码
3. 更新环境变量
4. 拉取新镜像
5. 重启服务
6. 运行数据库迁移
7. 烟雾测试
8. Slack通知

#### 4.2.2 Production环境

**触发条件**: 推送到main分支

**部署步骤**:
1. 创建GitHub Release（如果是tag）
2. 数据库备份
3. SSH连接到production服务器
4. 拉取最新代码
5. 蓝绿部署:
   - 启动新版本容器
   - 健康检查
   - 切换流量
   - 停止旧版本容器
6. 运行数据库迁移
7. 集成测试
8. 性能测试
9. 备份到S3
10. Slack通知

**回滚策略**:
- 部署失败时自动回滚到上一个版本
- 使用备用的docker-compose-rollback.yml

### 4.3 质量门禁

| 检查项 | 阈值 | 失败处理 |
|--------|------|----------|
| 代码格式化 | 100%通过 | 阻止构建 |
| 代码覆盖率 | >80% | 警告 |
| 漏洞扫描 | 高危=0 | 阻止构建 |
| 单元测试 | 全部通过 | 阻止构建 |
| 响应时间P95 | <1s | 警告 |

## 5. 监控和告警

### 5.1 监控指标

#### 5.1.1 基础设施指标

- **CPU使用率**: >80%告警
- **内存使用率**: >85%告警
- **磁盘使用率**: >85%告警
- **磁盘IO**: >80%告警
- **网络流量**: 异常波动告警

#### 5.1.2 应用指标

- **服务可用性**: !=1告警
- **HTTP错误率**: >5%告警
- **响应时间P95**: >1s告警
- **响应时间P99**: >2s告警

#### 5.1.3 数据库指标

- **MySQL连接数**: >80%告警
- **MySQL慢查询**: >10/s告警
- **Redis连接数**: >80%告警
- **Redis内存**: >80%告警

#### 5.1.4 业务指标

- **请求量**: <10/s告警（异常下降）
- **API错误率**: >1%告警
- **广告投放失败**: >100/s告警
- **预算消耗**: >80%告警
- **结算失败**: >10/s告警

### 5.2 告警分级

| 级别 | 严重性 | 响应时间 | 通知方式 |
|------|--------|----------|----------|
| Critical | 严重 | 5分钟 | 邮件+钉钉+企业微信 |
| Warning | 警告 | 30分钟 | 邮件+钉钉 |
| Info | 信息 | 不紧急 | 邮件 |

### 5.3 告警路由

```
告警 → AlertManager
    │
    ├─ 严重告警 → critical-receiver → 邮件+钉钉+企业微信
    ├─ 基础设施 → infra-receiver → 邮件+钉钉
    ├─ 应用 → app-receiver → 邮件+钉钉
    ├─ 数据库 → db-receiver → 邮件+钉钉
    ├─ 容器 → container-receiver → 邮件+钉钉
    ├─ 业务 → business-receiver → 邮件+钉钉
    └─ 日志 → log-receiver → 邮件+钉钉
```

### 5.4 告警抑制

- 如果服务宕机，抑制该服务的其他告警
- 如果节点宕机，抑制该节点上的所有告警
- 如果数据库宕机，抑制相关的应用告警

## 6. 安全配置

### 6.1 网络安全

- 所有服务运行在Docker私有网络（172.20.0.0/16）
- 仅Nginx暴露80/443端口
- 监控服务通过htpasswd认证
- 服务器防火墙配置

### 6.2 应用安全

- JWT密钥: 32字符以上
- 数据库密码: 强密码
- Redis密码: 强密码
- Sentry DSN: 错误追踪

### 6.3 SSL/TLS

- 协议: TLSv1.2, TLSv1.3
- 加密套件: 现代加密套件
- HSTS: max-age=31536000
- 自动续期: 每12小时检查

### 6.4 依赖安全

- Trivy漏洞扫描
- Snyk依赖扫描
- 定期更新依赖
