## 项目概述

本项目是一个面向国内市场的全媒体广告投放管理平台，支持抖音、快手、微信、微博、B站等主流媒体渠道的广告投放管理。

#### 2.3 用户角色定义（7种角色）

| 角色代码 | 角色名称 | 主要职责 |
|---------|---------|---------|
| SUPER_ADMIN | 超级管理员 | 系统最高权限，系统配置和用户管理 |
| ADMIN | 管理员 | 机构内的用户、账户、权限管理 |
| ADVERTISER | 广告主 | 拥有自己的广告账户，负责广告投放 |
| OPTIMIZER | 优化师 | 负责广告计划创建、优化、数据分析 |
| OPERATOR | 操作员 | 执行日常广告操作，权限受控 |
| VIEWER | 查看者 | 仅拥有查看权限，可查看数据和报表 |
| AUDITOR | 审计员 | 负责操作日志审计和合规检查 |

#### 2.4 技术栈建议

**后端技术栈**
- 开发语言: Python 3.10+
- Web框架: FastAPI
- ORM: SQLAlchemy 2.0
- 数据库: MySQL 8.0
- 缓存: Redis 7.0
- 消息队列: RabbitMQ / Kafka
- 任务调度: Celery / APScheduler

**前端技术栈**
- 开发框架: React 18 + TypeScript
- UI框架: Ant Design 5.x
- 状态管理: Redux Toolkit / Zustand
- 图表库: ECharts / Recharts
- 构建工具: Vite

**开发运维工具**
- 版本控制: Git + GitLab
- CI/CD: GitLab CI / GitHub Actions
- 监控: Prometheus + Grafana
- 日志: ELK Stack
- 链路追踪: Jaeger / SkyWalking

#### 2.5 数据库设计方案

**数据表清单**: 18张核心数据表

- 用户权限表: 6张（用户、角色、权限、组织等）
- 媒体账户表: 3张（渠道、账户、权限）
- 广告投放表: 4张（计划、组、创意、素材）
- 数据报表表: 2张（日报、实时）
- 预算风控表: 2张（预算配置、预警记录）
- 操作日志表: 1张

**分表策略**: 3张表按月分表
- dsp_report_daily
- dsp_report_hourly
- sys_operation_log

#### 2.6 非功能需求

- **性能**: 支持1000+并发用户，API响应时间<200ms
- **可用性**: 系统可用性≥99.9%
- **安全性**: HTTPS加密、JWT认证、RBAC权限、操作审计
- **可扩展性**: 支持水平扩展、读写分离、插件化渠道接入
- **可维护性**: 结构化日志、全链路监控、CI/CD自动化
