- **时间**: 2026-03-16 08:15
- **问题**: `ImportError: cannot import name 'Base' from 'models'`
- **原因**: `app/models/__init__.py` 中的循环导入
- **严重程度**: 🔴 高
- **影响**: 应用无法启动

---

## 问题分析

### 循环导入链

1. `app/models/__init__.py` 试图从 `. import Base`
2. 但 `Base` 定义在 `app/database.py` 中
3. 这导致启动时无法正确导入依赖关系

### 修复方案

**方案1**: 修改导入路径
```python
# 错误
from . import Base

# 正确
from app.database import Base
```

**方案2**: 添加缺失依赖
```python
# 缺少
from sqlalchemy import ForeignKey
```

## 经验教训

### 1. 循环导入检测

**教训**:
- ❌ 不要在包的 `__init__.py` 中使用相对导入
- ✅ 使用绝对导入路径：`from app.database import Base`
- ✅ 明确模块依赖关系

### 2. 依赖完整性

**教训**:
- ❌ 不要假设所有必需的包都已安装
- ✅ 在 `requirements.txt` 中明确列出所有依赖
- ✅ 在开发阶段验证所有导入

**教训**:
