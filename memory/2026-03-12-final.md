## 教训总结

### 关键教训

1. **Nginx 配置文件位置**
   - 不同系统/环境可能使用不同的配置文件位置
   - 必须验证实际加载的配置文件（使用 `nginx -T` 查看）
   - 修改后要验证是否生效

2. **Nginx 规则顺序**
   - 更具体的规则（`/admin/api/`）必须在通配规则（`/admin/`）之前
   - 使用 `^~` 前缀匹配优先级高于正则匹配
   - `proxy_pass` 的路径末尾斜杠会影响 URL 重写

3. **子路径部署配置**
   - Vite `base: '/admin/'`
   - React Router `basename="/admin"`
   - Nginx 反向代理 `/admin/api/` 和 `/admin/`
   - 前端 API `baseURL: '/admin/api'`
   - 四者必须保持一致

### 最佳实践

1. **部署前的检查清单**
