### 项目目标
交付一个外网可访问的商用级广告平台，具备以下功能：

- 广告管理系统（计划/组/创意）

- **前端**: React 18 + TypeScript + Ant Design 5.0
- **部署**: Docker + Docker Compose + Nginx
- **监控**: Prometheus + Grafana + Flower

## 💡 关键决策

### 技术选型
1. **FastAPI vs Django**: 选择FastAPI，因为异步性能更好，适合RTB竞价场景
2. **MySQL vs PostgreSQL**: 选择MySQL，因为更稳定，社区支持好
3. **Redis vs Memcached**: 选择Redis，因为支持更多数据结构，适合缓存和消息队列
4. **Docker vs K8s**: 选择Docker Compose，因为项目规模适中，Docker Compose更简单

### 项目结构
采用标准的FastAPI项目结构：
```
backend/
├── app/
│   ├── api/v1/          # API路由
│   ├── models/          # 数据模型（待完成）
│   ├── schemas/         # Pydantic模式（待完成）

│   └── main.py          # 主应用
```
